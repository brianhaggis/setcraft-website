Current release · 1.4.13

July 4, 2026 Current

Version 1.4.13

  • Woodstock, a new theme. A sixties flower-child field: warm tan with mod daisies that drift and bounce very slowly behind your work, over a pastel lava-lamp wash. A little lamp button by the wordmark turns the lava on or off, and Shrikhand lettering carries the headings. It settles to a still field if you have Reduce Motion on.
  • The Shire, a new theme. A warm hobbit-hole at dusk, with golden fireflies drifting through and a gold wordmark that catches a flash of light as you pass over it. This one takes the place of the old Big Hair look.
  • Synthwave trades its busy neon grid for a cleaner, deep twilight purple.

July 3, 2026

Versions 1.4.11–1.4.12

  • Typo is now one of the main themes instead of a hidden one. It sits near the top of the theme list, and its wordmark carries a small sun/moon switch that flips the whole look between light and dark.
  • Report a bug without leaving Setcraft. A tab on the right edge opens a short form where you describe the problem and attach a screenshot, and it sends straight to support. Needs an internet connection.
  • The Mystery themes shelf in Backstage now shows how to unlock each secret look, so there is a hint to chase instead of a blank slot.
  • Print buttons added to the Lyric Book and Chart Book in the Song Library, alongside Download.
  • Backstage now keeps you on the tab you were working in after an action (hiding, renaming, or deleting a band) instead of jumping back to the first tab.
  • Fixed a bug where a band whose name contained an apostrophe could not be deleted or renamed.

July 2, 2026

Version 1.4.10

  • Email your setlists, books, and charts. Every place with an AirDrop button now has an email button beside it, which opens a new message in your Mac's mail app with the PDF already attached. Covers setlists, the lyric and chart books, single-song lyrics and charts, the tech cue sheet, and the PRO report.
  • Song Library export buttons are grouped and labeled: a "Lyric Book" box with Download / AirDrop / Email, and a matching "Chart Book" box, so it is clear which book each button acts on.
  • What's New menu item added under Help (above Check for Updates), which opens the release notes for the running version.

July 1, 2026

Versions 1.4.2–1.4.9

  • Typo theme polish: the self-typing wordmark no longer nudges the nav links on each page load, and the current page's nav link now highlights clearly.
  • Wake Up theme: the CRT scanlines are softer and gently flicker.
  • First-launch welcome dialogs no longer overlap: the beta and trial explainers now appear one at a time instead of stacking.
  • Typo theme: the wordmark still types itself in, but it no longer nudges the nav links each time a page loads.
  • Wake Up theme: softened the CRT scanlines and gave them a subtle occasional flicker.
  • Add Multiple. A new button on the Songs page opens a grid where you can type in a batch of songs at once (title, short name, key, capo) or paste them straight from a spreadsheet. The grid grows as you go, and the whole batch imports in one step. Add lyrics, charts, and tech notes later by editing a song.
  • Short names now read in ALL CAPS everywhere they appear on screen, the way a stage label should. The Typo theme (below) and Diva keep their own casing.
  • New theme: Typo. A black-and-white typewriter look, set entirely in monospace and lowercase on warm paper, with a wordmark that types itself in.
  • Wake Up gets a CRT-terminal makeover: the wordmark and headings glitch, over faint scanlines.
  • The Songs selection bar no longer hides behind the trial banner when you scroll.
  • Sharpie mode (Rock Salt) prints cleanly. Capo lines and between-song notes no longer clip: Rock Salt gets taller slots so a title, its capo, and a wrapping note all fit, and the marker font's tall glyphs stop cropping at the top and bottom.
  • On the song library, selecting songs now tints the whole row, using the same color the setlist builder uses to highlight overlapping songs.
  • The sort arrow in the song library is now visible in every theme (it matches the column-header text color instead of the accent, which vanished on some themes like Yacht Rock).
  • In Backstage's Print/Export tab, the divider above Include lyric book now spans the full width, above the chart-book option too.
  • The Add / Edit Song dialog is wider, with the note fields beside the rest so the whole form fits without scrolling. If you close it with unsaved edits, it now asks whether to save, discard, or keep editing.
  • On the song library, clicking anywhere on a row selects it (except its buttons), and the selection bar stays put at the top of the list and follows you as you scroll, so it never jumps the page or leaves your actions out of reach.
  • On printed setlists, metaset subsongs now print at the same size as the between-song notes instead of towering over them.
  • Backstage is redesigned around a floating left sidebar and a wide, centered canvas, so most settings fit without scrolling. Print options move to their own Print/Export tab with a live sample preview, and unlocked secrets get their own tab.
  • Print options are now toggle switches that show what's on at a glance, with a clear "Show key / capo / notes / band name / venue / date / song numbers" set, plus new controls to hide the band name, venue, or date on a sheet. The venue shortens to fit and expands to its full name when there's room.
  • Sharpie mode now defaults to a scrawled "Rock Salt" marker (with Permanent Marker as an option), in natural title case with the key in parentheses.
  • The song library sorts with a Finder-style arrow showing the active column and direction.
  • A new Licenses menu (View / Manage License, Deactivate This Computer, Deactivate All Other Computers) in the menu bar.
  • Print defaults are shared with the print dialog: set them once in Backstage, or save your choices straight from the dialog. Cancelling a print never changes them.
  • The print dialog gains two options: Hide song numbers for a clean, unnumbered sheet, and Sharpie mode, which prints the setlist in a bold handwritten marker font like it was scrawled on the spot. Both show live in the preview.
  • Action buttons now carry their meaning in their color, consistent across every theme: green to save or create, blue to print or export, red to delete, orange to cancel. Upgrade buttons are green and inviting.
  • The print dialog's preview now shows the trial/free watermark exactly as it will print, with a note that a Pro or Road Warrior license removes it.
  • Fixed the bundled fonts not loading in the print preview, so Sharpie mode and the standard print font both render correctly on screen.
  • The free trial now marks printed and exported sheets with a light diagonal stamp, the same as the free version. Every feature stays open during the trial; a clean, gig-ready sheet is what a Pro or Road Warrior license unlocks.
  • Photo import shows the correct trial allotment: five scans a month, not twenty, and the counter now tracks as you use them.
  • The trial banner stays pinned below the top bar as you scroll, and its button reads "Upgrade to keep using all Pro features."
  • On the builder, the Print, cue sheet, and delete actions are right-aligned in their row.

June 30, 2026

Versions 1.4.0–1.4.1

  • The "See the tiers" button on the trial welcome screen now opens the license page as intended.
  • Pro and Road Warrior appear on the Account tab as side-by-side cards, each with a short description of what it includes, so the difference reads at a glance.
  • Clearer trial banner wording. During the trial you already have every feature, so the banner now points to keeping the full app after the trial rather than unlocking it.
  • New installs now get a free 30-day trial of the full app: every feature, no watermark, no card. After 30 days, Setcraft settles into the free version, and your library, shows, and setlists always stay on your Mac.
  • A small banner shows how many trial days remain, with a one-time welcome that explains how the trial works.
  • If a license is close to expiring, Setcraft now gives you a heads-up beforehand, so the full features never switch off unexpectedly. It is dismissable.
  • Photo import is fairer: a scan only counts against your monthly allotment when it actually reads something. A blurry photo that comes back empty no longer costs you.

June 29, 2026

Versions 1.2.52–1.3.0

  • New pricing model: two one-time tiers. Pro ($19.99) includes one artist profile; Road Warrior ($34.99) includes unlimited artist profiles. The older "additional artist profile" add-on is retired, and your artist limit now follows your tier.
  • Already on Pro and want more bands? Upgrade to Road Warrior from inside the app and pay only the difference, no need to buy again. Your library carries straight over.
  • Exporting a portable backup to move your whole library to another Mac is now part of Pro and Road Warrior. Restoring a backup stays free for everyone, including on a fresh install, and the automatic on-device snapshots are unchanged, so your data is always safe and never trapped.
  • Paid features (watermark-free exports, photo import, Bandsintown sync, the free song cap) now run through one consistent entitlement check, so they behave the same way everywhere.
  • When you build a setlist, you can now create a brand-new show right from the start window, without leaving for the Shows page.
  • Photo import can now turn a line it doesn't recognize into a new song on the spot. Enter the title and short name, your main artist is filled in for you (editable for covers), and it's added to your library as you import.
  • The action buttons on each show in the Shows list are easier to read.
  • The Print / Export dialog now shows a live preview of the page before you print or save, so you can see the layout and check your options first.
  • Added layout options in that dialog: a density choice (Comfortable, or Compact to fit more songs per column) and a sets-per-page choice (pair two sets on a page, or give each set its own page).
  • On printed setlists, the key now sits on the same line as the song title, with capo and other details in parentheses just beneath it.
  • Added a toggle at the top of the Shows page to switch between Upcoming, Past, and See All.
  • Fixed metaset (medley) subsong order not sticking. Reordering the songs inside a medley now saves and prints in the order you set.
  • Added a brief "working" indicator when AirDrop, Print, or Export is preparing a file, so it's clear the action is underway before the system window opens.
  • The Print and PRO Report dialogs now give every export button the same highlight, so none looks like the default.

  • Raised the song short-name limit from 12 to 15 characters.
  • Added a "Copy to Show" button on a saved show. Copy its setlist to a new or upcoming show and open it straight in the builder.
  • The Print, PDF, and AirDrop actions now appear as soon as a show is open in the builder, instead of waiting for the first save.
  • Fixed metaset songs disappearing when you added or rearranged songs in the main set after building the medley.

June 18, 2026

Versions 1.2.14–1.2.16

  • Fixed license activation from the first-launch welcome screen. Entering a key and email on the onboarding "Activate" step failed with "Could not reach the activation server," even though the same key worked once you skipped to the free version and activated from Backstage. The onboarding guard was redirecting the activation request to the wizard page before it reached the license routes, so the response came back as HTML and the screen misread it as a connection failure. The /api/activate and /api/license/reclaim routes are now allowed to run during onboarding.
  • The onboarding wizard's "Pick a look" step now keeps its Back and Next buttons in view. The theme tiles were large enough that the full set pushed Next off the bottom of the window, which made it look like the buttons in the preview behind the picker were how you advanced. The tiles are now compact so everything fits, and the preview behind the picker no longer shows its (non-clickable) nav links, so there is nothing back there that looks like a way forward.
  • The top navigation links now show on the dashboard too, instead of disappearing there. Having them vanish on the landing page was more confusing than helpful.
  • The "Backstage" navigation item now carries a gear icon, so its settings purpose reads at a glance even if the name's backstage metaphor does not land (for example for users whose first language is not English).
  • Backstage Artists tab: the Primary and Active badges next to a profile name now sit in a tidy row, evenly spaced from the name and aligned at the same height.
  • The Bandsintown sync window now links to a short "How to connect Bandsintown" guide for getting your API key, and the free-version sync and Photo Import prompts open straight to the Account & License tab so a license key is one stop away. Links to outside pages now open in your browser instead of inside the app.
  • The demo band's name no longer shows a rename (pencil) button on the dashboard. The demo is a read-only sample, so its name was never editable; the button is gone to match.
  • Account & License: you can now enter an additional-artist-profile add-on key right on the Account & License tab. It used to live only inside the Manage Artists dialog, which is not where most people went looking for it.
  • When you have a paid artist seat you have not used yet, Backstage now points it out: a note on the Account & License tab and a small count badge on the Artists tab, so an activated-but-uncreated profile does not sit there unnoticed. The badge reads clearly on every theme.

June 8, 2026

Versions 1.2.0–1.2.5

  • Licensing cleanup: the app no longer bundles any copyleft components. PDF rasterization moved from PyMuPDF (AGPL) to PDFium, and the HEIC/HEVC libraries (which pulled in GPL-licensed x265) were dropped from the app, since that work only ever runs on the server. The shipped binary is now permissive/LGPL only.
  • Added an Open-Source Licenses page, reachable from the About panel, the Help menu, Backstage, and the website.
  • Help > About Setcraft now opens the standard About panel (it was an AppKit call made off the main thread, so it silently did nothing).
  • Removed the macOS window tab bar. Setcraft is a single-window app, so the default tabbing (the View > Show Tab Bar item and the empty tab strip) only added confusion.
  • Tutorial copy fixes: the overview's closing card now explains how to reach the demo band (via the Artist Switcher) rather than implying you are already in it; the songs "Type" step no longer claims covers print the original artist (they do not); the Backup tutorial highlights the Backup tab.
  • Lyric and chart books now print, not just PDF export. The desktop Print path honors the lyric/chart book checkboxes instead of dropping them.
  • Photo Import no longer shows "Setlist created" and "Review Matches" before you import anything (a desktop-only gate bug).
  • Backstage: the Account & License tab shows license status and the number of activated additional artist profiles; the Artists tab is wider so profile rows with long band names no longer wrap.
  • The Shows list now has a PRO Report link on each show with a setlist.
  • The dashboard rename (pencil) icon follows the theme color.
  • Tutorials: corrected the set cap (four, not unlimited), the default builder view (two sets, not one), and setlist saving (manual Save, not automatic); fixed the song filter list and the print-dialog wording. Added Lyrics, Charts, Shows filtering, and a Save step, a new Backup & Restore tutorial, and a data safety stop on the overview. Removed the redundant Run Setup Wizard entry.
  • Navigation and page labels are now consistently title-cased ("Build Setlist", "Photo Import") so they read correctly on themes whose fonts are not all-caps.
  • Automatic updates via Sparkle: the app checks the signed appcast feed (getsetcraft.com/appcast.xml) and can download, install, and relaunch a new version itself. Manual check from Help, Check for Updates.
  • Also carries the interim lightweight update check, which only runs on builds that do not have Sparkle, so there is never a double prompt.
